# Hackathon Project Specification: Multi-Agent Crop Intelligence Pipeline

## 1. Project Title & Executive Summary

* **Proposed Title:** Agrospheric AI: A Multimodal Multi-Agent Pipeline for Precision Phytopathology and Risk Mitigation.
* **Executive Summary:** Current agricultural AI tools are fundamentally siloed—offering either isolated crop disease image classification or basic tabular weather alerts. This project breaks that limitation by building an integrated, multi-agent intelligence pipeline. It fuses unstructured computer vision data (leaf pathology) with structured environmental metrics (soil composition and climate tracking). Organized as a deterministic multi-agent graph, the system converts raw, complex data streams into localized, explainable, and actionable intervention strategies for farmers.

---

## 2. The Problem Statement (The "Why")

Modern agriculture faces a dual challenge: rapid climate volatility and accelerating crop pathology. While AI has made strides in both areas, existing solutions suffer from three critical bottlenecks that prevent adoption and limit hackathon winning potential:

* **The Modality Gap:** Computer vision apps detect a disease *after* visible damage occurs, but they lack context. A weather app predicts drought or humidity spikes but doesn't know the health history of the crop. Without fusing these modalities, predictions are reactive rather than proactive.
* **The Black-Box Dilemma:** Farmers and agronomists mistrust raw AI outputs. Simply stating "Your crop has Late Blight" without showing *why* or *where* causes a lack of user trust.
* **The Accessibility Barrier:** High-tech PDF reports are useless to rural farmers who face language barriers or low literacy rates. Solutions must be localized, verbalized, and highly practical.

---

## 3. Technical Challenges to Tackle (The "How")

To secure maximum points for **Technical Complexity** and **Execution**, the architecture must explicitly solve these engineering hurdles:

### A. Multimodal Data Fusion

The system must ingest and synchronize two radically different data streams under a unified pipeline wrapper:

* **Unstructured Visual Stream:** Processing RGB images of crop leaves using a pre-trained, fine-tuned **Vision Transformer (ViT)** (`vit-base-patch16-224`).
* **Structured Tabular Stream:** Processing numerical vectors (Temperature, Humidity, Rainfall, Soil Nitrogen [N], Phosphorus [P], Potassium [K], and Moisture levels) using an optimized **XGBoost** classifier.

### B. State Management & Deterministic Agent Orchestration

Moving past fragile, linear LLM prompts, the orchestration layer must utilize graph-based agent design (**LangGraph** or **CrewAI**). The core technical challenge is managing global state across multiple specialized agents, enforcing strict JSON schemas for inter-agent communication, and routing conditional tasks dynamically.

### C. Explainable AI (XAI) Integration

Every analytical model must expose its decision-making parameters:

* **ViT:** Extracting self-attention maps or Grad-CAM heatmaps to visually isolate the infected regions of the leaf.
* **XGBoost:** Running local **SHAP (SHapley Additive exPlanations)** interpretations to quantify exactly how much an environmental parameter (e.g., a sudden 15% drop in soil moisture) contributed to the risk factor.

---

## 4. Non-Technical & Impact Constraints

To win the **Viability** and **Real-World Impact** judging criteria, the project must handle these production realities:

* **Human-In-The-Loop (HITL) Safety Gate:** AI cannot blindly prescribe chemical applications. If the Vision Node returns a disease classification confidence score below a threshold (e.g., less than 75%), the state graph must halt autonomous execution, flag the case as "Ambiguous," and route it to an agronomist review dashboard interface.
* **Vernacular Localization & Inclusivity:** The final output must bypass technical jargon. A specialized translation agent must convert the structural action plan into the farmer’s regional language and automatically synthesize a clear audio broadcast (`gTTS` or similar lightweight text-to-speech engine).
* **Resource-Constrained Optimization:** The models must run efficiently. We utilize open-weights, compact models capable of running on consumer-grade or edge-compute hardware rather than relying on massive, expensive proprietary API ecosystems.

---

## 5. Architectural Flow & Solution Blueprint

The system is compartmentalized into three independent, decoupled nodes:

| Node | Input Data | Core Technology | Primary Output |
| --- | --- | --- | --- |
| **1. Vision Node** | Leaf Image (JPEG/PNG) | Hugging Face ViT + Attention Extraction | JSON: `disease_class`, `confidence_score`, `attention_matrix` |
| **2. Analytical Node** | Weather/Soil Vector | XGBoost + SHAP | JSON: `14_day_risk_pct`, `top_3_environmental_stressors` |
| **3. Orchestration Layer** | Combined JSON outputs | LangGraph / CrewAI + Open LLM | Final PDF Report, Localized Text, and Audio Broadcast (.mp3) |
